Evaluating Accuracy and Bias in Machine Learning Models

A self-directed research project investigating how AI models learn to recognise road signs — and where they fail. Using Google's Teachable Machine platform, Lola trained a neural network on road sign images and then stress-tested it with 26 images the model had never seen before, including blurry shots, distant signs, and busy urban backgrounds.

What I Did

  • Selected two road sign classes with distinct shapes and colours: Stop signs (octagon, red) vs. Curve signs (diamond, yellow)
  • Trained a neural network using Google's Teachable Machine with 13 images per class as a baseline
  • Stress-tested the model with 26 unseen "edge case" images — blurry, distant, and set against complex backgrounds
  • Recorded the model's prediction and confidence level for every single image to identify success and failure patterns

What I Found

The model achieved 92.3% accuracy — but the failures were revealing. In one test, the AI was 100% confident that a blurry Stop Sign was a Curve Sign. This highlighted a key danger: AI can be "confidently wrong." The project also revealed how bias enters machine learning when training data isn't diverse enough — a model trained only on dark images becomes blind to signs in bright light.

Real-World Implications

Self-Driving Cars

Real autonomous vehicles require 99.9999% accuracy ("six nines") — our 92.3% would cause accidents daily.

Sensor Fusion

Real cars solve camera blur using LiDAR, Radar, and HD Maps as redundant checks — no single sensor is trusted alone.

AI Bias

Diverse training data is essential. A model that only sees one type of image fails to generalise to the real world.
